### Content Insights

- Docker can replace any tool installation, not just servers. Any language runtime, build tool, or CLI can be aliased this way. I used to be obsessed with resetting my pc because i liked how clean it felt. This method allows me to keep my PC clean while still using a plethora of convoluted tools.
- docker is a build tool not just a runtime. i could see that people might be at level 2 (docker is used to run commands, but still relies on something else being built first) and think thats it, but level 3 shows that you can use docker to not only be the runtime, but create the entire environment.
- This corrects my understanding of docker files in that they can be multi-step (where i thought previously you would need a docker compose), but a docker compose is more to do with multi-image systems that rely on eachother for volumes or order of activation.
- Running `docker compose down` completely removes the containers compared to `docker compose stop` which halts them temporarily. Running `down` notably does not remove volumes, the data is safe.
