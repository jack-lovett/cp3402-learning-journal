### Content Insights

- "you might think you need a database call, but do you really? Could that database call have already happened and generated a static file". does the information im showing need to be dynamic? if so then an api call is appropriate, but if the information being displayed is not going to change, you can fetch the information first, store that, and serve it statically. You could integrate these api calls into the build process itself via docker or something.
- cdns act as a protective layer as users are exposed to their CDN node, not your servers directly, this makes it more difficult to pinpoint your actual server and perform attacks.
- cdns are more efficient in terms of the UX as it reduces loading times and theoretically can serve at the same speed no matter where you are on the planet.
- JAMstack = JavaScript + APIs + Markup. Gutenberg = WordPress's block editor.
- Serving a static site through a Worker made routing feel more intentional. I had to think about what should be reachable, what should 404, and how paths resolve, instead of assuming "the server will handle it".

### Career/Employability/Learning Insights

- I noticed I learn infrastructure tooling best by doing short loops: make one change, run the deploy/dev command, and only then read the docs again with a specific question in mind. When I tried to read everything first, I didn't retain much.
- This week felt closer to real work than I expected: small configuration details had a lot of impact, and fixing was about being systematic and verifying assumptions step by step.
